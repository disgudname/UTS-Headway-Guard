Route Destinations

UTS routes are loops — TransLoc has no direction/destination concept, so any "heading to…" text has to be picked by hand. Click a stop below to mark it as a destination landmark and give it a short label (the real stop name is usually too long for signage). /v1/transloc/stop_arrivals then labels every arrival with the next marked landmark ahead on that route's loop, wrapping around when needed.

CAT already resolves a real destination automatically from each route's pattern data, so tagging is optional here — use it to override a pattern's raw headsign with a friendlier rider-facing label, e.g. marking a stop as "Hospital" or "Willoughby". Pick a pattern below to walk its actual stop sequence and tag any stop along it. Landmarks are tied to the physical stop, not the pattern, so a label you set here shows up in every other pattern that happens to pass through the same stop. Patterns are point-to-point (not loops like UTS), so the label only applies to stops after the tagged one — it doesn't wrap around. The very last stop of a pattern's timeline is a special case: a bus ending its run there has no "next stop" to point to, so it gets its own separate terminus label instead (shown at the bottom of that pattern's timeline) — keyed to the pattern rather than the stop, since the same physical terminus can send different patterns off in different directions afterward.
